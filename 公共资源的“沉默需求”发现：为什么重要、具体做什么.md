## 公共资源的“沉默需求”发现：为什么重要、具体做什么

### 一、为什么重要：三个层面的价值

#### 1. 理论层面：填补从“抱怨”到“需求”的研究空白

当前研究大多停留在**识别抱怨**的层面——知道有人在抱怨某个问题，但很少系统性地将这些抱怨**转化为“未满足的公共服务需求”** 。这是一个关键的断裂带。

已有研究关注到了社交媒体在公共事务中的作用。有学者以湖北省“东湖社区”网络问政平台**2426条**“问题诉求—方案回应”对为对象，采用分词、聚类和共现网络分析等方法，确定了公众的问题诉求重点并划分了不同类别。也有研究基于人民网“领导留言板”**2011至2023年**的用户留言数据，构建了大数据驱动的居民核心需求动态识别与预测框架，覆盖城市建设、医疗、教育、交通等多个领域。

但这些研究的**数据来源主要是政务平台**（人民网留言板、政府网站等），而非日常社交媒体。政务平台上的诉求是**主动、显性、有意识**的表达——用户本就抱着“我要投诉”的目的去留言。而社交媒体上的表达是**自然、非结构化、无意识**的——居民在抱怨“楼下太吵”时，并没有意识到自己在向政府提需求。**从这种“无意识抱怨”中提取“有意识需求”** ，正是“沉默需求发现”的独特价值所在。

有研究指出，居民诉求表达渠道正逐渐向微博、抖音等网络平台，以及微信群、QQ群等社交媒介迁移，而传统治理模式在应对高频、细微、易发酵的社群矛盾时，普遍存在**响应滞后、程序繁琐、辐射面窄**等结构性问题。这进一步说明，利用社交媒体数据进行主动需求感知，是对传统被动响应模式的重要补充。

#### 2. 实践层面：解决传统调研的三大痛点

传统的公共服务需求调研（问卷、访谈、市民热线）存在明显局限：

| 痛点       | 传统方式                       | “沉默需求发现”的解决方案        |
| -------- | -------------------------- | -------------------- |
| **样本偏差** | 问卷只覆盖愿意配合的人群，大量“沉默的大多数”被忽略 | 社交媒体覆盖全人群，被动采集，无主动干预 |
| **时效滞后** | 问卷从设计到分析周期长，无法捕捉动态变化       | 实时采集，可追踪需求的时空演变      |
| **表达失真** | 问卷中的回答受问题设置影响，不一定反映真实状态    | 社交媒体表达是自发的、真实的日常记录   |

正如有学者所强调的，社交媒体平台已成为事实上的公共论坛，公民在这里表达对城市服务的抱怨和不满。而**舆情监测分析研判**正是借助大数据、自然语言处理、知识图谱等技术，将散落在社交平台上的海量碎片信息转化为**可感知、可量化、可操作**的治理依据。“沉默需求发现”正是这种理念在公共服务领域的精准落地。

#### 3. 政策层面：契合城市治理的转型方向

当前智慧城市建设普遍存在**技术驱动倾向**，忽视了真实的居民需求，这与“**以人为本**”的原则相悖。传统的自上而下需求收集方式难以捕捉公共需求的**动态性和异质性**。而社交媒体数据为这一挑战提供了新的解决方案。

在实践层面，已有城市做出了探索。深圳市通过“随手拍”模式，鼓励市民通过社交媒体给公共设施提建议，已累计主动发现**17万条**民生诉求线索。有研究者提出，社交媒体中关于社区公共服务设施的评价内容具有**多维度、多层次、多形式**的特征，这类数据可以为设施的**使用后评估体系**建立提供支持。

### 二、具体做什么：从数据到决策的完整链路

#### 核心逻辑

**居民不会发帖说“我要公园”，但他们会抱怨“楼下太吵”、“孩子没处玩”。** 这个项目的核心，就是**把抱怨翻译成需求**。

#### 技术路线（三阶段）

**Lab 1：数据采集——找到“抱怨在哪里”**

选定一个城市（如北京或你所在的城市），围绕公共设施设定关键词进行爬取：

- **设施类关键词**：公园、充电桩、人行道、天桥、路灯、垃圾桶、公交站、公共厕所、健身器材、图书馆、菜市场等
- **动作类关键词**：太吵、没地方、不够、坏了、没人管、不方便、要是能……就好了
- **平台选择**：微博（数据量大、带地理位置标签）、小红书（生活化表达丰富）

注意：微博和小红书的帖子常带有**地理标签（POI）** ，这是后续做空间聚合的关键。

**Lab 2：情感/观点分析——把“抱怨”翻译成“需求”**

这一阶段的核心不是简单的积极/消极分类，而是**意图识别 + 实体抽取**：

- **意图分类（三分类）** ：
  
  - **抱怨类**：“楼下广场舞太吵了”“这条路晚上黑漆漆的”
  - **建议类**：“要是这里有个充电桩就好了”“能不能在这修个天桥”
  - **直接询问类**：“附近哪有24小时图书馆”“哪里有免费的健身器材”

- **命名实体识别（NER）** ：从文本中提取被抱怨的**设施名称**（如“路灯”“充电桩”）和**地点**（如“望京SOHO附近”）

已有成熟研究可以借鉴。例如，有研究利用NLP技术（包括命名实体识别、情感分析和多标签文本分类）处理城市公民抱怨。也有研究采用BERTopic模型进行主题建模，以及用**BERT**进行文本分类。这些都是可以直接复用的技术方案。

- **需求映射规则**（关键创新点）：
  - “路灯坏了” → “需要路灯维修”
  - “广场舞太吵” → “需要噪音治理或公共活动空间规划”
  - “孩子没处玩” → “需要儿童游乐设施”
  - “晚上不敢走这条路” → “需要道路照明或治安巡逻”
  - “找不到充电的地方” → “需要公共充电桩”

这些映射规则就是“从抱怨到需求”的**翻译词典**，也是这个项目区别于一般情感分析的核心所在。

**Lab 3：生成式决策支持——产出可执行的报告**

将前两步的分析结果输入大模型，生成结构化的决策支持产品：

- **需求热力图**：标注哪些区域抱怨最密集、需求最迫切
- **设施缺口清单**：
  - “望京地区缺乏公共充电桩（23条相关抱怨）”
  - “回龙观地区路灯损坏严重（17条相关抱怨）”
  - “通州某小区缺乏儿童游乐设施（12条相关抱怨）”
- **建设优先级排序**：按抱怨频次、影响人群、紧迫程度打分排序

有研究已证明，通过NLP技术自动检索、分类和处理公民投诉，可以将非结构化的噪声文本转化为政府部门可行动的洞察。这正是Lab 3要实现的目标。

#### 成果形式

最终汇报时呈现一份**“城市公共服务‘沉默需求’发现报告”** ，包含：

1. **需求热力图**（用地图标注需求热点区域）
2. **设施缺口清单**（按设施类型和区域归类）
3. **优先级排序与建议措施**（哪些问题最紧迫、建议如何解决）
4. **方法论说明**（关键词设计依据、分类模型设计、需求映射规则）

### 三、为什么适合这个课程项目

| 评估维度        | 说明                                           |
| ----------- | -------------------------------------------- |
| **研究gap明确** | 从“抱怨”到“需求”的转化链条缺乏系统研究；现有研究多依赖政务平台数据，而非日常社交媒体 |
| **技术路线成熟**  | 爬虫、BERT分类、NER、大模型生成，每一步都有现成工具和文献支撑           |
| **四天可完成**   | 三个Lab与课程进度完全对齐，每天完成一个模块                      |
| **数据可得性高**  | 微博/小红书公开数据充足，不依赖突发事件                         |
| **成果可视化强**  | 热力图+需求清单，汇报时直观有力                             |
| **社会价值突出**  | 直接服务城市治理，符合“以人为本”的智慧城市建设方向                   |

已有研究证实了这类方法的可行性。卡尔加里大学的一项研究表明，**利用NLP从大规模社交媒体文本中提取智慧城市需求**是可行的，当地官员可以利用这一系统加速决策过程。西安建筑科技大学的研究团队也证明，基于大数据的需求识别与预测框架在各类需求主题上的**拟合优度（R²）达到了0.93到0.96**。

**“沉默需求发现”不是要发明新算法，而是要发明一个新视角——把社交媒体上散落的抱怨，翻译成城市管理者可以执行的行动清单。** 这正是社会计算“从数据到决策”的核心理念的完美体现。
